## From Linear SVM to Non-Linear SVM — Why We Moved Forward

In Experiment 6, Linear SVM achieved excellent results (~97%) on the Breast Cancer dataset because that data was **largely linearly separable** in feature space. However, real-world data is often **not linearly separable** — no straight line (or hyperplane) can cleanly divide the classes.

**The problem with linear SVM on non-linear data:**
- A linear kernel can only draw straight decision boundaries.
- On circularly arranged data (like `make_circles`), a linear SVM will fail — it cannot separate an inner circle from an outer circle with a line.
- This demonstrates the need for **non-linear kernels** that can create curved, complex boundaries.

**The solution — Kernel functions:** By projecting data into a higher-dimensional space (using the kernel trick), non-linear boundaries in original space become linear in the higher-dimensional space. The RBF (Radial Basis Function) kernel is particularly powerful for this.

## Steps Performed

**Cell 2 — Generate & Visualize Dataset:**
Generated 100 samples with `noise=0.2` (adds random perturbation) and `factor=0.5` (inner circle radius relative to outer). The scatter plot clearly shows two concentric rings — red points (Class 0) form the outer ring, blue points (Class 1) form the inner ring. A straight line cannot separate them.

**Cell 3 — Preprocessing:**
No missing values (synthetic data). Split 80/20. Applied `StandardScaler` — fit on train only, transform on test (no data leakage). Training: 80 samples, Testing: 20 samples.

**Cell 4 — Linear SVM as Baseline:**
Trained `SVC(kernel='linear')`. As expected, performance is poor — the linear decision boundary plot shows it trying to draw a straight line through circular data, failing to separate the classes correctly. Accuracy is noticeably low (often ~50–60%) — close to random guessing.

This confirms: **linear SVM is unsuitable for non-linearly separable data.**

**Cell 5 — RBF SVM:**
Trained `SVC(kernel='rbf')`. The decision boundary plot shows a **curved boundary** that correctly separates the two circles. Accuracy jumps dramatically — typically **90–95%**. The RBF kernel maps the data into infinite-dimensional space, making the circular boundary linear in that space.

**Cell 6 — Polynomial and Sigmoid Kernels + Comparison Table:**
- **Polynomial (degree=3):** Creates a curved boundary. Performs reasonably well but may not capture perfect circles as well as RBF.
- **Sigmoid:** Generally weaker on this dataset — decision boundary is less well-defined.

Comparison table:
| Kernel | Accuracy |
|--------|----------|

This clearly demonstrates that **kernel choice is critical** for non-linear data.

**Cell 7 — GridSearchCV Hyperparameter Tuning:**
Grid searched over `kernel` (rbf, poly, sigmoid), `C` (0.1, 1, 10, 100), `gamma` (scale, auto, 0.01, 0.1), and `degree` (2, 3, 4). 5-fold cross-validation used. Best parameters printed — typically RBF with C=10 and gamma=0.1 or similar. Best CV accuracy is printed.

**Cell 8 — Evaluate Best Model:**
Tested the best model on the test set. Confusion matrix (heatmap) and classification report printed. Decision boundary of the best model visualized — showing a clean circular separation. Final accuracy typically matches or slightly exceeds the non-tuned RBF result.

## Key Concepts Learned

**Non-Linear Data:** Data where classes cannot be separated by a straight line or hyperplane. Real-world examples include image classification, medical diagnoses, and pattern recognition.

**RBF (Radial Basis Function) Kernel:**  
$$K(x_i, x_j) = \exp\left(-\gamma \|x_i - x_j\|^2\right)$$  
Maps data into infinite-dimensional space. The `gamma` parameter controls the width of the Gaussian — large gamma means tight fit to training points (risk of overfitting), small gamma means smoother boundary.

**Polynomial Kernel:**  
$$K(x_i, x_j) = (\gamma \langle x_i, x_j \rangle + r)^d$$  
Maps to polynomial feature space of degree `d`. Good for data with polynomial relationships.

**Sigmoid Kernel:**  
$$K(x_i, x_j) = \tanh(\gamma \langle x_i, x_j \rangle + r)$$  
Mimics a neural network's activation function. Less commonly used with SVM.

**The Kernel Trick:** Instead of explicitly computing coordinates in high-dimensional space, kernels compute the **dot product in that space directly** — making the computation tractable even in infinite dimensions.

**C vs gamma (RBF kernel):**
- `C`: Regularization — controls margin width vs misclassification penalty.
- `gamma`: Controls influence radius of each training point — high gamma = each point influences only its immediate neighborhood.

**GridSearchCV with Cross-Validation:** Tests all combinations of hyperparameters using k-fold cross-validation. Prevents overfitting to the validation set. Reports the best combination based on CV accuracy.
